// WHAT'S WORKING

Design Token System

50+ CSS custom properties organized by surface role (canvas, note, toolbar, overlay, dialog). Dual-theme with dark readings that feel authored — deep graphite canvas with cool-white text, not a simple invert.

Interaction Loop

Paste creates notes at cursor position. Drag repositions in canvas space. Double-click opens CodeMirror editor. Debounced auto-save to IndexedDB. The product's core promise is delivered cleanly.

Performance Discipline

Transform/opacity-only animations, lazy-loaded vim extension, CSS background-image dot grid, debounced DB writes. Zero layout thrash. Fast time-to-interactive.

// PRIORITY ISSUES
P0

No undo for destructive actions

Delete is permanent with zero confirmation. A single Backspace keypress removes a note from state and IndexedDB instantly. No undo buffer, no trash, no toast. The design principle says "Undo beats confirm" — especially for delete, move, and edit operations.

FIX: Add undo stack to Zustand store (snapshot before destructive ops) or a 5-second toast with "Undo" button. Direct store refactor — no new dependencies needed.
P1

Touch targets below minimum

Toolbar buttons are 34×34px (WCAG recommends 44×44px). Context menu color swatches are 20×20px — nearly impossible to tap accurately on a phone. Mobile use is obstructed by tiny hit areas even though the product conceptually works on touch devices.

FIX: Expand toolbar button containers to 44×44px minimum. Increase swatch size to 28px with adequate spacing. Use padded containers rather than visually enlarging icons.
P1

Contrast failure on placeholder and muted text

Light mode: placeholder (#94a3b8 on white) ≈ 3.0:1 — fails WCAG AA for any text size. Muted (#64748b on white) ≈ 4.5:1 — marginal for normal text, fails for small text below 14px. Dark mode: placeholder (#40526a on #141720) ≈ 3.2:1 — fails AA.

FIX: Darken light-mode placeholder to #6b7280, muted to #4b5563. Lighten dark-mode placeholder to #5a7090. Direct token edits in globals.css — two-location change.
P2

No focus trap in modal dialogs

Search overlay, command palette, and edit dialog are marked aria-modal="true" but don't constrain Tab focus. A keyboard user tabbing past the last focusable element leaves the overlay and lands on hidden canvas UI behind it. Focus is lost, not trapped.

FIX: Add a focus-trap hook or wrapper. Standard pattern: Tab from last element cycles to first, Shift+Tab from first cycles to last. ~30 lines of code.
P2

No screen reader feedback on state changes

When notes are created, deleted, duplicated, or pinned, there is no aria-live announcement. Screen reader users receive no confirmation that their action was successful. The visually-hidden assistive text channel is missing entirely.

FIX: Add an aria-live="polite" region to the canvas (visually hidden) and push status messages on create, delete, duplicate, and pin state changes.
P2

Notes don't adapt to narrow viewports

Notes have a fixed 320px width. On a 320px-wide iPhone SE viewport, a note fills the entire screen horizontally with no breathing room. While the canvas is pannable, notes don't scale or constrain to available width.

FIX: Clamp note width to min(320px, calc(100vw - 32px)). Deeper pass: container queries for note content reflow on small containers.
